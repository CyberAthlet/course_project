## 1. Постановка задачи

Необходимо разработать симулятор добычи газа для системы из трёх скважин, работающих на один общий пласт. Симулятор должен:

- рассчитывать дебит и давления в каждой точке системы;
- моделировать истощение пласта во времени;
- учитывать работу дожимной компрессорной станции (ДКС);
- калиброваться по промысловым данным.

### Схема системы

```
┌─────────────────────────────────────────────────────────────────────────┐
│  ПЛАСТ (общий)                                                          │
│  P_res(t), V_res, T_res                                                 │
└──────────┬──────────────────┬──────────────────┬───────────────────────┘
           │                  │                  │
    [Скважина 1]        [Скважина 2]       [Скважина 3]
    k₁,h₁,rₑ₁,rw₁       k₂,h₂,rₑ₂,rw₂     k₃,h₃,rₑ₃,rw₃
    L₁, H₁, D₁          L₂, H₂, D₂        L₃, H₃, D₃
    P_bhp,1              P_bhp,2            P_bhp,3
           │                  │                  │
           └──────────────────┴──────────────────┘
                              │
              q_ext ─────► [Куст / Манифолд]
              (сторонний       P_man(t)
               газ)              │
                     [Шлейф]  L_s, D_s, δ_s
                   q_total = q₁+q₂+q₃+q_ext
                              │
                     [ДКС]   P_out = CR · P_in
                              │
                  [Магистральный газопровод]
                         P_line = const
```

Давление в системе убывает слева направо: пласт → забой → устье → шлейф → ДКС → магистраль.

## 2. Математические основы

**Допущение:** температура во всей системе (пласт, скважины, шлейф, ДКС) принимается постоянной и равной пластовой: $T = T_\text{res} = \text{const}$. Это позволяет задавать температуру один раз и не передавать её в каждый расчёт.

### 2.1 Уравнение состояния реального газа

Реальный газ описывается уравнением:

$$PV = Z n R T$$

где $P$ — давление [Па], $V$ — объём [м³], $n$ — количество вещества [моль], $R = 8.314$ Дж/(моль·К), $T$ — температура [К], $Z$ — коэффициент сверхсжимаемости (Z-фактор).

Для идеального газа $Z = 1$. Для реального газа $Z \neq 1$ и зависит от $P$ и $T$.

**Z-фактор** рассчитывается по модифицированному уравнению GERG-91 согласно ГОСТ 30319.2-96 (раздел 3.2.3). Метод расчёта подробно разобран в задании `hw2.ipynb` — используйте реализацию оттуда.

#### Плотность газа

Из уравнения состояния:

$$\rho = \frac{P M}{Z R T}$$

где $M$ — молярная масса газа [кг/моль], $P$ в Па.

#### Стандартные условия

В промысловой практике объёмы газа приводят к стандартным условиям:

$$P_\text{std} = 101325 \text{ Па}, \quad T_\text{std} = 293.15 \text{ К}$$

Плотность при стандартных условиях:

$$\rho_\text{std} = \frac{P_\text{std}  M}{Z_\text{std}  R  T_\text{std}}$$

где $Z_\text{std} \approx 1$ (при стандартных условиях).

#### Фактор объёма газа $B_g$

$B_g$ показывает, какой объём занимает 1 стандартный м³ газа при пластовых условиях:

$$B_g = \frac{\rho_\text{std}}{\rho_\text{res}} = \frac{P_\text{std}  Z_\text{res}  T_\text{res}}{P_\text{res}  Z_\text{std}  T_\text{std}}$$

При $Z_\text{std} \approx 1$:

$$\boxed{B_g(P) = \frac{P_\text{std}  Z(P)  T}{P  T_\text{std}}}$$

### 2.2 Вязкость газа

Динамическая вязкость $\mu(P)$ [сП] задана в виде таблицы в файле `interp_data.csv`. Для промежуточных значений давления используется **линейная интерполяция**.

Формула кусочно-линейной интерполяции для точки $x_p$, попадающей между узлами $x_i$ и $x_{i+1}$:

$$y_p = y_i + \frac{y_{i+1} - y_i}{x_{i+1} - x_i} (x_p - x_i)$$

**Требование:** реализовать интерполятор без использования NumPy/SciPy внутри класса — только стандартные структуры Python.

### 2.3 Пластовая модель

#### Закон Дарси для радиальной фильтрации

Скорость фильтрации газа в пористой среде задаётся законом Дарси:

$$v = -\frac{k}{\mu} \frac{dP}{dr}$$

где $k$ — проницаемость [Д], $\mu$ — вязкость газа, $r$ — радиус.

Для стационарного радиального течения записывается уравнение неразрывности:

$$\frac{1}{r}\frac{d}{dr}\!\left(r \frac{k}{\mu}\frac{dP}{dr}\right) = 0$$

Интегрируя от $r_w$ (радиус скважины) до $r_e$ (радиус контура питания) и переходя к объёмному дебиту при пластовых условиях $q_\text{res}$ [м³/сут], получаем:

$$q_\text{res} = \frac{2\pi k h}{\mu \ln(r_e/r_w)} (P_\text{res} - P_\text{bhp})$$

В промысловых единицах (мД, м, атм, м³/сут) вводится коэффициент $\beta$:

$$\boxed{q_\text{std} = \frac{\beta  k  h}{\mu  \ln(r_e/r_w)} (P_\text{res} - P_\text{bhp})}$$

где $\beta = 0.00852702$ [ст.м³/(сут·мД·м·атм)] — коэффициент перевода единиц, $q_\text{std}$ — дебит в стандартных условиях [ст.м³/сут]. Параметры $k$, $h$, $r_e$, $r_w$ — свойства конкретной скважины (у разных скважин они могут различаться).

Формулу удобно записать через коэффициент продуктивности $C_i$ скважины $i$:

$$q_{\text{std},i} = C_i (P_\text{res} - P_{\text{bhp},i}), \quad C_i = \frac{\beta  k_i  h_i}{\mu  \ln(r_{e,i}/r_{w,i})}$$

Кривая $q_\text{std}(P_\text{bhp})$ при фиксированном $P_\text{res}$ называется **кривой притока (IPR — Inflow Performance Relationship)**.

#### Материальный баланс пласта

Все три скважины разрабатывают **один общий пласт**. Уравнение сохранения массы газа в закрытом пласте объёмом $V_\text{res}$:

$$\frac{d(m_\text{res})}{dt} = -\sum_{i=1}^{3} \dot{m}_i$$

Масса газа в пласте: $m_\text{res} = \rho_\text{res}  V_\text{res}$, массовый дебит $i$-й скважины: $\dot{m}_i = q_{\text{std},i} \rho_\text{std}$.

Подставляя $m_\text{res} = \rho_\text{res} V_\text{res}$ и $\dot{m}_i = q_{\text{std},i} \rho_\text{std}$, получаем

$$\rho_\text{res}^{(t+1)} = \rho_\text{res}^{(t)} - \frac{\rho_\text{std}}{V_\text{res}}\sum_i q_{\text{std},i} \Delta t$$

а затем переходим к давлению через уравнение состояния ($\rho = PM / ZRT$).

После дискретизации по времени с шагом $\Delta t$ [сут] (все величины вычисляются при $P = P_\text{res}^{(t)}$):

$$\boxed{P_\text{res}^{(t+1)} = P_\text{res}^{(t)} - \frac{Z(P_\text{res}^{(t)})  \rho_\text{std}}{\rho(P_\text{res}^{(t)})} \frac{\sum_i q_{\text{std},i}}{V_\text{res}} \Delta t}$$

### 2.4 Гидравлика трубопровода. Уравнение Дарси–Вейсбаха

Уравнение применяется как к **НКТ скважин**, так и к **шлейфу**. Перепад давления вдоль трубы длиной $L$ складывается из двух слагаемых:

$$\Delta P = \underbrace{\lambda \frac{L}{D} \frac{\rho v^2}{2}}_{\text{трение}} + \underbrace{\rho  g  H}_{\text{гидростатика}}$$

где:
- $\lambda$ — коэффициент гидравлического сопротивления (безразмерный),
- $D$ — внутренний диаметр трубы [м],
- $\rho$ — средняя плотность газа в трубе [кг/м³],
- $v$ — средняя скорость газа [м/с],
- $H$ — вертикальная глубина [м] (для горизонтальной трубы $H = 0$).

В рабочих единицах [атм]:

$$\boxed{\Delta P [\text{атм}] = \frac{1}{101325}\left(\lambda \frac{L}{D} \frac{\rho v^2}{2} + \rho  g  H\right)}$$

#### Скорость газа в трубе

Объёмный расход газа при пластовых условиях: $q_\text{res} = q_\text{std} B_g$ [м³/сут].

Средняя скорость в поперечном сечении трубы:

$$\boxed{v = \frac{4 q_\text{std} B_g}{\pi D^2 86400}} \quad [\text{м/с}]$$

#### Число Рейнольдса

$$Re = \frac{\rho  v  D}{\mu}$$

где $\mu$ в [Па·с] = $\mu$ [сП] / 1000.

#### Коэффициент трения. Уравнение Колбрука–Уайта

Для турбулентного течения в трубах с шероховатостью стенок применяется **неявное уравнение Колбрука–Уайта**:

$$\boxed{\frac{1}{\sqrt{\lambda}} = -2 \lg\!\left(\frac{\delta}{3.7 D} + \frac{2.51}{Re \sqrt{\lambda}}\right)}$$

где $\delta$ — абсолютная шероховатость стенки [м]. Для стальных труб типичное значение $\delta = 0.000046$ м $= 0.046$ мм. **Внимание:** в формулах $\delta$ подставляется в **метрах**.

Уравнение **неявное** относительно $\lambda$ и решается итерационно. Схема решения методом простых итераций:

1. Начальное приближение: $\lambda^{(0)} = 0.02$
2. На каждом шаге $n$:

$$\lambda^{(n+1)} = \left[-2 \lg\!\left(\frac{\delta}{3.7 D} + \frac{2.51}{Re \sqrt{\lambda^{(n)}}}\right)\right]^{-2}$$

3. Повторять до $|\lambda^{(n+1)} - \lambda^{(n)}| < 10^{-6}$

Альтернативно — передать уравнение в `scipy.optimize.fsolve`, записав его в виде $f(\lambda) = 0$.

#### Давление на забое через давление на устье

Для НКТ скважины $i$:

$$P_\text{bhp,i} = P_\text{man} + \Delta P_\text{tube,i}(q_i)$$

Кривая $P_\text{bhp}(q)$ при фиксированном $P_\text{man}$ называется **кривой подъёма (VLP — Vertical Lift Performance)**.

Для шлейфа ($H = 0$):

$$P_\text{man} = P_\text{in,DCS} + \Delta P_\text{shlyf}(q_\text{total})$$

где $q_\text{total} = q_1 + q_2 + q_3 + q_\text{ext}$ — суммарный расход через шлейф, включая сторонний газ.

---

### 2.5 Дожимная компрессорная станция (ДКС)

ДКС повышает давление газа, поступающего по шлейфу, до давления магистрального газопровода.

**Степень сжатия:**

$$CR = \frac{P_\text{out}}{P_\text{in}}$$

Граничное условие: $P_\text{out} = P_\text{line}$ — давление в магистрали (константа).

Отсюда давление на входе в ДКС:

$$\boxed{P_\text{in,DCS} = \frac{P_\text{line}}{CR}}$$

При $CR = 1$ ДКС не работает, $P_\text{in,DCS} = P_\text{line}$.

**Замечание:** при увеличении $CR$ давление на входе в ДКС снижается, что уменьшает $P_\text{man}$, увеличивает депрессию на пласт и, следовательно, суммарный дебит $q_\text{total}$.

---

### 2.6 Нахождение рабочей точки системы

Рабочая точка — это набор дебитов и давлений, при котором вся система находится в гидравлическом равновесии. Неизвестные: $q_1,  q_2,  q_3,  P_\text{man}$.

Каждая скважина характеризуется собственным коэффициентом продуктивности $C_i = \dfrac{\beta  k_i  h_i}{\mu  \ln(r_{e,i}/r_{w,i})}$ и собственной НКТ с перепадом давления $\Delta P_{\text{tube},i}$. Поэтому дебиты скважин в общем случае различны.

Система четырёх нелинейных уравнений:

$$\begin{cases}
q_1 = C_1 \bigl(P_\text{res} - P_\text{man} - \Delta P_\text{tube,1}(q_1)\bigr) \\
q_2 = C_2 \bigl(P_\text{res} - P_\text{man} - \Delta P_\text{tube,2}(q_2)\bigr) \\
q_3 = C_3 \bigl(P_\text{res} - P_\text{man} - \Delta P_\text{tube,3}(q_3)\bigr) \\
P_\text{man} = \dfrac{P_\text{line}}{CR} + \Delta P_\text{shlyf}(q_1 + q_2 + q_3 + q_\text{ext})
\end{cases}$$

где $q_\text{ext}$ [ст.м³/сут] — расход стороннего газа, поступающего на манифолд (задаётся как параметр ДКС, см. раздел 3.8). Через шлейф и ДКС проходит суммарный объём $q_\text{total} = q_1 + q_2 + q_3 + q_\text{ext}$.

Записывается в виде вектора невязок $\mathbf{F}(\mathbf{x}) = \mathbf{0}$, где $\mathbf{x} = [q_1, q_2, q_3, P_\text{man}]^\top$, и решается численно с помощью `scipy.optimize.fsolve` или `scipy.optimize.root`.

**Геометрическая интерпретация:** для каждой скважины строится кривая IPR и кривая VLP на одном графике $q$ vs $P_\text{bhp}$. Их пересечение — рабочая точка данной скважины при известном $P_\text{man}$.

### 2.7 Динамический симулятор

Явная схема дискретизации по времени:

$$\text{дано:}\; P_\text{res}^{(0)}, \quad \text{задать:}\; N_\text{days},\; \Delta t \;[\text{сут}]$$

На каждом шаге $t$:

1. Решить систему уравнений (2.6) → получить $q_1^*, q_2^*, q_3^*, P_\text{man}^*$
2. Если $q_i^* < 0$ — установить $q_i^* = 0$ (скважина условно закрывается).
3. Обновить пластовое давление по формуле материального баланса (2.3):

$$P_\text{res}^{(t+1)} = P_\text{res}^{(t)} - \frac{Z(P_\text{res}^{(t)}) \rho_\text{std}}{\rho(P_\text{res}^{(t)})} \frac{q_\text{total}^*}{V_\text{res}} \Delta t$$

4. Вычислить накопленную добычу:

$$G_p^{(t+1)} = G_p^{(t)} + q_\text{total}^* \Delta t$$

где $q_\text{total}^* = \sum_i \max(0,  q_i^*)$ — суммарный дебит скважин куста (без стороннего газа).

### 2.8 Калибровка по промысловым данным

Промысловые данные содержатся в файле `adapt_gdi_11-2025.csv` (столбцы: дата, $q$, $P_\text{res}$, $P_\text{bhp}$, $P_\text{thp}$).

Задача калибровки — подобрать параметры модели так, чтобы симулятор воспроизводил измеренные дебиты. В упрощённой постановке оценивается один параметр — коэффициент продуктивности $C$.

**Целевая функция:**

$$\text{RMSE}(C) = \sqrt{\frac{1}{N}\sum_{i=1}^{N}\bigl(q_i^\text{sim}(C) - q_i^\text{fact}\bigr)^2}$$

Минимизируется с помощью `scipy.optimize.minimize` (метод `'Nelder-Mead'` или `'L-BFGS-B'`).

**Метрики качества** после калибровки:

$$R^2 = 1 - \frac{\sum(q_i^\text{fact} - q_i^\text{sim})^2}{\sum(q_i^\text{fact} - \bar{q}^\text{fact})^2}$$

$$\text{RMSE} = \sqrt{\frac{1}{N}\sum(q_i^\text{fact} - q_i^\text{sim})^2}$$


**Требования:**
- `Reservoir` — это «бак»: хранит текущее состояние пласта (`ResProps`) и рассчитывает материальный баланс.
- `p2` — новое пластовое давление по формуле материального баланса (раздел 2.3). Принимает суммарный дебит всех трёх скважин.
- После вызова `p2` состояние `resprops.P` **не изменять** — это делает `FieldSimulator`.
- Параметры продуктивности ($k$, $h$, $r_e$, $r_w$) в `Reservoir` **не хранятся** — они принадлежат скважинам (см. раздел 3.7).

---

### 3.6 Модуль `pipe.py`

```
Pipe(L: float, D: float, roughness: float, fluid: Fluid, vertical_depth: float = 0.0)
    # L [м], D [м], roughness (δ) [м], vertical_depth (H) [м]

    dp(P: float, q: float) -> NodeState  # состояние элемента, q [ст.м³/сут]
    # остальные методы — на усмотрение студента
```

**Требования:**
- `dp` — основной метод. Вычисляет перепад давления по формуле Дарси–Вейсбаха (раздел 2.4) с учётом `vertical_depth` и возвращает `NodeState` с заполненными полями. Для шлейфа создаётся экземпляр `Pipe(..., vertical_depth=0.0)`.
- Коэффициент трения $\lambda$ рассчитывать по уравнению Колбрука–Уайта (раздел 2.4). Для ламинарного режима ($Re < 2300$) использовать формулу Пуазейля: $\lambda = 64 / Re$.
- Один класс `Pipe` используется и для НКТ скважин, и для шлейфа.
- Построение кривой VLP и другие вспомогательные методы реализовать самостоятельно.

---

### 3.7 Модуль `well.py`

```
Well(fluid: Fluid, k: float, h: float, re: float, rw: float, pipe: Pipe = None)
    # k [мД], h [м], re [м], rw [м]
    # pipe — объект Pipe, описывающий НКТ данной скважины (задаётся при сборке симулятора)

    q(P_res: float, P_bhp: float) -> float  # дебит [ст.м³/сут]
    # остальные методы — на усмотрение студента
```

**Требования:**
- `Well` объединяет приток газа из пласта (закон Дарси, раздел 2.3) и подъём по НКТ (`Pipe`).
- `q(P_res, P_bhp)` — дебит скважины по формуле Дарси при заданном пластовом давлении $P_\text{res}$ и забойном давлении $P_\text{bhp}$.
- Коэффициент продуктивности $C$, построение кривой IPR и другие вспомогательные вычисления реализовать самостоятельно.
- `pipe` (НКТ) используется симулятором для расчёта перепада давления в стволе (см. `Pipe.dp`).

---

### 3.8 Модуль `compressor.py`

```
DCS(CR: float, P_line: float, q_ext: float = 0.0)
    # CR — степень сжатия (≥ 1.0), P_line [атм]
    # q_ext — расход стороннего газа, поступающего на манифолд [ст.м³/сут]

    P_in() -> float   # давление на входе в ДКС [атм]
```

**Требования:**
- При `CR = 1.0` возвращать `P_line` (станция отключена).
- `CR` и `q_ext` должны быть изменяемыми атрибутами (для анализа чувствительности).
- Сторонний газ $q_\text{ext}$ поступает на манифолд и проходит через шлейф и ДКС вместе с газом куста. В материальный баланс пласта он **не входит**.

---

### 3.9 Модуль `simulator.py`

```
FieldSimulator(
    reservoir: Reservoir,
    wells: list,      # список из 3 объектов Well
    shlyf: Pipe,
    dcs: DCS
)
    solve(P_res: float) -> dict[str, NodeState]
    # возвращает словарь NodeState по именам элементов:
    # 'well_1', 'well_2', 'well_3', 'shlyf', 'dcs'

    run(N_days: int, dt: float = 1.0) -> DataFrame
    # колонки: t [сут], P_res [атм], P_man [атм],
    #          q1, q2, q3, q_total [ст.м³/сут], Gp [тыс.ст.м³]
```

**Требования:**
- `solve` формирует вектор невязок $\mathbf{F}(\mathbf{x})$ размером 4 и вызывает `scipy.optimize.fsolve` или `scipy.optimize.root`. Для каждой скважины дебит рассчитывается через `Well.q(P_res, P_bhp)`, а перепад давления в стволе — через `well.pipe.dp(...)`. Рекомендуемое начальное приближение: $q_{1,2,3}^{(0)} \approx 500$ ст.м³/сут, $P_\text{man}^{(0)} \approx P_\text{line}/CR + 5$ атм. В четвёртом уравнении учитывать $q_\text{ext}$ из объекта `dcs`.
- После решения: если $q_i < 0$ — установить $q_i = 0$ (скважина закрывается).
- `solve` возвращает `dict[str, NodeState]` с ключами `'well_1'`, `'well_2'`, `'well_3'`, `'shlyf'`, `'dcs'`.
- `run` реализует явную схему из раздела 2.7. Параметр `dt` задаёт шаг по времени (по умолчанию 1 сут). На каждом шаге вызывает `solve`, затем обновляет `reservoir.resprops.P`. В материальный баланс пласта входит только дебит скважин ($q_1 + q_2 + q_3$), без стороннего газа. Метод должен выводить прогресс симуляции в консоль (например, каждые 10 шагов или с помощью библиотеки `tqdm`).
- `run` всегда выполняет ровно $N_\text{days}$ шагов. При запуске использовать `dt = 1`.
